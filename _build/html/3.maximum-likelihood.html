Maximum Likelihood Estimation#

Notation for Estimates#

So far, our notation for our model has implicitly assumed that we have the whole popluation at our disposal and know the values for the parameters. However, in reality, we will usually only have a sample from that population and will need to estimate the parameter values. Because the distinction between the true value and an estimate is quite important, we need some new notation to make this clear.

Traditionally, we denote an estimate by placing a “hat” on top of the corresponding Greek letter. For instance, if \(\beta_{1}\) corresponds to the true value of the regression slope in the population, we would denote an estimate of \(\beta_{1}\) from a sample as \(\hat{\beta}_{1}\). The importance of this distinction then goes further than just indicating that \(\hat{\beta}_{1}\) is simply a “guess” based on some data. Because \(\hat{\beta}_{1}\) will change for every new sample we take, its value will shift around. As such, \(\hat{\beta}_{1}\) is a random variable. This is important, because \(\beta_{1}\) is constant and \(\hat{\beta}_{1}\) is random. As such, \(\hat{\beta}_{1}\) has a distribution with an expectation and a variance. We will get into the specifics of this later, but for the moment it is worth noting that an expression such as \(E\left(\hat{\beta}_{1}\right)\) and \(\text{Var}\left(\hat{\beta}_{1}\right)\) make complete sense because our estimates are random, whereas \(E\left(\beta_{1}\right)\) and \(\text{Var}\left(\beta_{1}\right)\) do not make sense because the true population values are constant. So keep an eye on those little hats, they make quite a big difference to the meaning.

Alternative Notation for Estimates

Some authors like to use the Latin alternatives to the Greek letters to denote estimates. We will not be doing this, but it is worth knowing about to make sense of notation you may see in textbooks or the literature. In this scheme, the estimate of \(\beta_{1}\) would be \(b_{1}\). So, the true population model would be

\[ y_{i} = \beta_{0} + \beta_{1}x_{i} + \epsilon_{i}, \]

whereas as an estimated model based on data would be

\[ y_{i} = b_{0} + b_{1}x_{i} + e_{i}. \]

You can decide which of these you prefer, but we will be using “hats” in all our notation going forward.

Calculating Estimates with Maximum Likelihood#

set.seed(666)

y          <- rnorm(n=100, mean=50, sd=1)
mu.ests    <- seq(40,60,length.out=1000) # 1000 possibilities for mu
likelihood <- rep(0,length(mu.ests))

for (i in 1:length(mu.ests)){
    likelihood[i] <- prod(dnorm(y, mean=mu.ests[i], sd=1))
}

plot(mu.ests,likelihood, type="l")

max.idx <- which.max(likelihood)
mu.ests[max.idx]
